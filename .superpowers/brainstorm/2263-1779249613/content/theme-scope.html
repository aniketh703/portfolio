Theme scope — how wide should dark/light mode go?

This shapes the architecture. Pick the one that matches your vision.

A

Whole site toggle

A single sun/moon icon in the navbar switches everything — home, archive, brand page, ProjectView — between light and dark. One persistent preference saved to localStorage.

Light: stone-50 base Dark: stone-950 base
B

ProjectView only — always dark

The main site stays on its current warm light theme. ProjectView becomes permanently dark (stone-950 background, stone-50 text) — it already feels like a separate immersive space, so lean into it. No toggle needed.

Site: light (unchanged) ProjectView: dark always
C

ProjectView toggle — independent of main site

The main site stays light. Inside ProjectView, a small toggle lets viewers switch between a light reading mode and a dark immersive mode. Preference persists per-session. Best for long case study reads.

Site: light (unchanged) ProjectView: user toggles

My lean: Option B or C. Your main site has a strong, intentional warm-light identity. Introducing a full-site dark mode risks muddying that. ProjectView is already a separate "world" — making it dark by default (B) or toggleable (C) amplifies that contrast dramatically without touching the rest of the site.
