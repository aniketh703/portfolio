## Remaining Issues by Priority

### 🔴 Critical

#### 1. SPA — No Pre-Rendered HTML
**Problem:** GitHub Pages serves a blank `<div id="root">` to crawlers. Googlebot *can* execute JavaScript and index SPAs, but it processes them in a delayed second wave (days later, not real-time). Other crawlers (Bing, DuckDuckGo, LinkedIn preview, Slack unfurl) cannot execute JS at all and will see nothing.

**Impact:** LinkedIn/Slack shares show broken previews despite OG tags in react-helmet (because the shell HTML doesn't have them — only the JS-rendered DOM does). Search indexing is slower.

**Fix:** Add static pre-rendering via `react-snap` or migrate to Astro/Next.js.
```bash
npm install -D react-snap
# Add "postbuild": "react-snap" to package.json scripts
```
`react-snap` crawls the SPA after build and saves pre-rendered HTML files — works with GitHub Pages, zero infrastructure changes.

**Effort:** 2–4 hours | **Impact:** Very High

---

#### 2. OG Tags in `<head>` vs JS-rendered
**Problem:** Because react-helmet-async writes OG tags into the live DOM via JS, social media crawlers (LinkedIn, Slack, Twitter/X) that don't execute JS pick up the *shell* `index.html` OG tags, not the page-specific ones. LinkedIn shares of `/work/1` will show the homepage title, not "GEETA APP | Aniketh Vustepalle".

**Fix:** Pre-rendering (same as above) or a serverless OG image function (Cloudflare Worker/Vercel Edge).

**Effort:** 4–8 hours | **Impact:** High (social sharing CTR)

---

### 🟠 High

#### 3. Content Depth — Project Case Studies
**Problem:** Every project description is 1–2 sentences. Google's Helpful Content guidelines reward pages with substantial, original content (400+ words). Short project cards can't rank for any keyword beyond the brand name.

**Target keywords you're missing:**
- "Bhagavad Geeta app design" — no competition, you'd rank #1
- "career guidance platform UI design"
- "enterprise SaaS dashboard redesign"
- "HITL AI interface design"
- "UI/UX designer Hyderabad portfolio"

**Fix:** Write 400–800 word case studies for your top 4 projects. Include: problem statement, research process, design decisions, results, tools used. Each case study page (e.g., `/work/1`) can rank independently.

**Effort:** 2–4 hours per project | **Impact:** Very High (long-term)

---

#### 4. `lastmod` Dates in Sitemap
**Problem:** All sitemap dates are hardcoded to `2026-05-25`. Google uses `lastmod` as a crawl priority hint. Stale dates signal nothing changed.

**Fix:** Either automate `lastmod` updates in your deploy script, or just update manually when you publish new work.

**Effort:** 15 min | **Impact:** Medium

---

#### 5. No `loading="lazy"` on Project Cover Images in Archive/Home
**Problem:** Project images in the grid load eagerly by default. Above-fold images should load eagerly; below-fold should be lazy.

Check: `Home.jsx` has `loading="lazy"` on project images ✅. Verify `Archive.jsx` `ProjectThumb` component also sets it on the `<img>` tag.

**Fix (Archive.jsx line 12):**
```jsx
<img
  src={project.coverImage}
  alt={project.title}
  onError={() => setImgFailed(true)}
  className="w-full h-full object-cover"
  loading="lazy"
/>
```

**Effort:** 5 min | **Impact:** Core Web Vitals (LCP/FID)

### 🟡 Medium

#### 7. OG Image Size
**Problem:** `public/og-image.jpg` is **261KB**. Social platforms cache this, but it's 30% over the recommended 200KB ceiling.

**Fix:** Re-export from Figma at 80% JPEG quality or run through `squoosh.app`. Target: 120–160KB.

**Effort:** 5 min | **Impact:** Page load speed, social share load time

---

#### 8. No `alt` Text on Fallback Images in ProjectView
**Problem:** When `ProjectThumb` falls back to `<GenerativeArt>`, there's no accessible description of the project thumbnail. `aria-hidden` is now set on the SVG, but the parent `<img>` fallback path doesn't exist — the whole thing is just a visual decoration with no text alternative.

**Fix:** In `ProjectThumb`, ensure the fallback `GenerativeArt` is wrapped in something with `role="img" aria-label={project.title}`:
```jsx
return (
  <div role="img" aria-label={`${project.title} — decorative thumbnail`}>
    <GenerativeArt id={project.id} color={project.color} />
  </div>
);
```

**Effort:** 10 min | **Impact:** Accessibility (WCAG 1.1.1)

#### 10. Blog on External Domain
**Problem:** The blog lives at `aniketh-blog.appwrite.network`. Any SEO value from blog posts (backlinks, content authority) flows to that domain, not your portfolio domain.

**Long-term fix:** Self-host the blog at `aniketh703.github.io/portfolio/blog` using a static site generator (Astro, Eleventy). Each blog post compounds your domain authority and could rank for "AI product design" related searches.

**Effort:** 8–16 hours (migration) | **Impact:** High (long-term, compound)

---

### 🟢 Low / Informational

#### 11. `emptyOutDir: false` in vite.config.js
**Problem:** Build output directory is never cleared. Over many builds, stale files accumulate in `dist/`. This can cause cached old pages to be served.

**Fix:** Change to `emptyOutDir: true` (the Vite default) unless you have a specific reason to preserve old build output.

**Effort:** 1 min | **Impact:** Low (build hygiene)

---

#### 12. No `sitemap.xml` Reference in `<head>`
**Problem:** The sitemap URL is in `robots.txt` ✅ but not discoverable via a `<link>` tag in HTML.

This is minor — Google finds it via robots.txt. No action needed.
